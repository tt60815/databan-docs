### 2.1 order_manager.py — 自動下單管理器

**檔案：** `ib_dashboard/order_manager.py`（327 行）

**功能：** 策略引擎偵測到信號後，透過 thread-safe Queue 將下單請求傳遞給 IB 線程執行。

#### 資料結構

```python
@dataclass
class OrderRequest:
    strategy_id: int        # 策略 ID
    symbol: str             # 'TSLA'
    action: str             # 'BUY' / 'SELL'
    quantity: int           # 股數
    direction: str          # 'long' / 'short'
    order_type: str         # 'entry' / 'exit'
    exit_reason: str        # 'stop_loss' / 'take_profit' / 'exit_condition' / ...
    entry_price: float      # 進場時的參考價格

@dataclass
class ManagedPosition:
    strategy_id: int
    symbol: str
    direction: str
    quantity: int
    entry_price: float      # IB fill 成交價
    entry_time: datetime
    entry_order_id: int     # IB orderId
    stop_loss_price: float
    take_profit_price: float
```

#### 跨線程通訊模型

```
Strategy Thread (Databento bars)          IB Thread (_ib_thread)
         │                                       │
         │  submit_order(OrderRequest)            │
         │         │                              │
         │    ┌────▼────┐                        │
         │    │  Lock   │  ← 檢查 _inflight      │
         │    └────┬────┘                        │
         │         │                              │
         │    ┌────▼──────────────┐              │
         │    │   queue.Queue    │──────────────▶ process_queue()
         │    │  (thread-safe)   │              │       │
         │    └──────────────────┘              │  ┌────▼──────┐
         │                                      │  │ _execute_  │
         │                                      │  │  order()   │
         │                                      │  │            │
         │                                      │  │ ib.place   │
         │                                      │  │  Order()   │
         │                                      │  └────┬──────┘
         │                                      │       │
         │                                      │  ┌────▼──────┐
         │                                      │  │ _on_fill() │
         │                                      │  │ callback   │
         │                                      │  └────┬──────┘
         │                                      │       │
         │                                      │       ▼
         │                                      │  DB: ib_trades
```

#### 安全機制

| 機制 | 實作 | 說明 |
|------|------|------|
| 全域開關 | `_config['enabled']` | API 可即時啟停交易 |
| 重複防護 | `_inflight` set + Lock | 同一策略不重複下單 |
| 已持倉防護 | `_positions` dict | 同策略不重複進場 |
| 最大持倉 | `max_total_positions=5` | 最多同時持有 5 個策略的倉位 |
| 小量下單 | `default_shares=10` | 每筆 ~$2,500（TSLA） |
| IB 斷線 | re-queue | 未連線時放回 Queue 等重連 |
| 訂單取消 | `_on_cancelled` | 清除 inflight 狀態 |

#### 核心方法

```python
class OrderManager:
    def submit_order(self, req: OrderRequest)       # 策略線程呼叫（thread-safe）
    def process_queue(self)                          # IB 線程呼叫（每 1 秒）
    def _execute_order(self, req: OrderRequest)      # ib.placeOrder(Stock, MarketOrder)
    def _on_fill(self, trade, fill)                  # IB fill 回調
    def _handle_entry_fill(self, req, trade)         # 建立 ManagedPosition → 寫 DB
    def _handle_exit_fill(self, req, trade)          # 計算 PnL → 更新 DB
    def _record_entry(self, pos: ManagedPosition)    # INSERT INTO ib_trades
    def _record_exit(self, pos, exit_price, ...)     # UPDATE ib_trades SET exit_*
    def get_all_positions(self) → dict               # API 查詢用
    def get_trades_from_db(self, limit=50) → list    # 查詢交易紀錄
    def set_ib(self, ib)                             # IB 連線注入
```

### 2.2 strategy_engine.py — 即時策略評估引擎

**檔案：** `ib_dashboard/strategy_engine.py`（628 行）

**功能：** 將回測的策略狀態機（IDLE → WATCHING → HOLDING）搬到即時環境，收到每根 1m bar 時評估策略條件並觸發下單。

#### 狀態機

```
┌────────┐   Entry 條件滿足   ┌──────────┐   Target 條件確認   ┌──────────┐
│  IDLE  │ ───────────────→  │ WATCHING │ ─────────────────→  │ HOLDING  │
│ 等待中 │                   │  觀察中  │                      │  持倉中  │
└────────┘                   └──────────┘                      └──────────┘
    ▲                            │                                  │
    │      Close 條件觸發        │                                  │
    └────────────────────────────┘                                  │
    │                                                               │
    │          3 種出場觸發                                          │
    └───────────────────────────────────────────────────────────────┘
              ①止損/止盈  ②均高買/均低賣  ③條件出場
```

#### StrategyInstance 資料結構

```python
@dataclass
class StrategyInstance:
    strategy_id: int
    symbol: str               # 'TSLA'
    entry_tf: str             # '5m'
    observe_tf: str           # '15m'
    direction: str            # 'long' / 'short'
    exit_mode: str            # 'exit_condition' / 'prev_high'

    # 條件配置
    entry_conditions: list    # [{field, operator, value}, ...]
    watch_conditions: list    # [{timeframe, conditions: [...]}, ...]
    target_conditions: list   # [{field, operator, value}, ...]
    close_conditions: list    # [{timeframe, conditions: [...]}, ...]
    exit_conditions: list     # [{timeframe, conditions: [...]}, ...]

    # 止損/止盈（預設 5%/5%）
    price_exit: dict          # {stop_loss: {enabled, percent}, take_profit: {enabled, percent}, priority}
    avg_hl_mode: dict         # {enabled: bool}

    # 執行狀態
    state: StrategyState      # IDLE / WATCHING / HOLDING
    entry_price: float
    entry_time: datetime
    signals: list             # 最多 50 筆信號紀錄
```

#### HOLDING 狀態三優先級出場

```python
# _evaluate() 中 HOLDING 狀態的出場邏輯：

# ═══ 優先級 1：止損/止盈（價格觸發）═══
# Long 止損：current_low  <= entry_price × (1 - stop_loss_pct / 100)
# Long 止盈：current_high >= entry_price × (1 + take_profit_pct / 100)
# Short 反向
# priority = "stop_loss_first" → 先檢查止損（預設）
# priority = "take_profit_first" → 先檢查止盈

# ═══ 優先級 2：均高/均低出場 ═══
# Long:  entry_tf 的 avg_low_sell == 1 → 出場
# Short: entry_tf 的 avg_high_buy == 1 → 出場

# ═══ 優先級 3：條件出場 ═══
# 遍歷 exit_conditions → check_conditions() → 滿足則出場
```

#### 核心流程

```python
def on_bar(self, symbol, bar_1m):
    """每收到一根 1m bar 觸發"""

    # 1. 每 5 分鐘自動 reload 策略（保留舊狀態）
    if time.time() - self._last_reload > 300:
        self.reload_strategies()

    # 2. 聚合到高 TF（1m → 5m/15m/1h ...）
    self.aggregator.on_bar(symbol, bar_1m)

    # 3. 計算需要的 TF 指標（有快取，bars 數量不變不重算）
    for tf in needed_tfs:
        bars = self.aggregator.get_all_bars(symbol, tf)
        if len(bars) >= BUFFER_BARS:  # 30
            indicators = calculate_indicators(bars)

    # 4. 對每個策略評估狀態機
    for strat in symbol_strategies:
        self._evaluate(strat, entry_row, observe_row)
```

#### 信號類型

| 信號 | 意義 | 狀態轉換 |
|------|------|----------|
| `ENTRY_SIGNAL` | 進場條件觸發 | IDLE → WATCHING |
| `CLOSE_CANCEL` | 觀察期間取消 | WATCHING → IDLE |
| `TARGET_ENTRY` | 目標確認，下進場單 | WATCHING → HOLDING |
| `EXIT` | 出場（含原因 + PnL） | HOLDING → IDLE |

### 2.3 tf_aggregator.py — 即時 K 棒時間框架聚合器

**檔案：** `ib_dashboard/tf_aggregator.py`（218 行）

**功能：** 將 Databento 即時 1m bar 聚合為 3m/5m/15m/30m/1h/2h/3h/4h K 棒，所有 TF 從 9:30 ET 開盤時間對齊。

#### 聚合邏輯

```
收到 1m bar (例：10:03 TSLA)
  │
  ├─ 解析時間 → 轉換到 NY 時區
  ├─ mins_since_open = (bar_time - 9:30) in minutes
  ├─ 過濾盤前/盤後（只處理 9:30-16:00）
  │
  └─ 對每個 TF（3m, 5m, 15m, 30m, 1h, 2h, 3h, 4h）：
      │
      ├─ bar_index = mins_since_open // tf_minutes
      │   例：10:03 → mins=33 → 5m bar_index=6
      │
      ├─ 若 bar_index 改變（新 TF K 棒開始）：
      │   └─ 上一根 TF K 棒 → 存入 bars deque（maxlen=200）
      │
      └─ 更新當前 TF K 棒：
          ├─ open  = 第一根 1m 的 open
          ├─ high  = max(所有 1m 的 high)
          ├─ low   = min(所有 1m 的 low)
          ├─ close = 最新 1m 的 close
          └─ volume = sum(所有 1m 的 volume)
```

#### 記憶體結構

```python
_data = {
    'TSLA': {
        '5m': {
            'current': {ts, open, high, low, close, volume},  # 正在構建
            'current_index': 6,
            'current_date': date(2026, 3, 12),
            'bars': deque([...], maxlen=200)  # 已完成 K 棒
        },
        '15m': { ... },
        '1h': { ... },
    }
}
```

### 2.4 indicators.py — 即時技術指標計算

**檔案：** `ib_dashboard/indicators.py`（227 行）

**功能：** 對聚合後的 K 棒計算技術指標，純 pandas/numpy 輕量實作，最少需要 30 根 K 棒暖機。

#### 計算的指標完整列表（36 個欄位）

| 分類 | 欄位 | 計算 |
|------|------|------|
| **MACD** | `dif` | EMA(close,12) - EMA(close,26) |
| | `dea` | EMA(dif, 9) |
| | `macd_hist` | (dif - dea) × 2 |
| **KDJ** | `k` | RSV 的 SMA（init=50） |
| | `d` | K 的 SMA（init=50） |
| | `j` | 3K - 2D |
| **Heikin-Ashi** | `ha_open/close/high/low` | 平滑 OHLC |
| **趨勢計數** | `dif_trend` | DIF 連續上升 +N / 下降 -N |
| | `dea_trend`, `k_trend`, `d_trend`, `j_trend`, `macd_hist_trend` | 同上 |
| **交叉信號** | `dif_cross` | +1=金叉 / -1=死叉 / 0=無 |
| | `kd_cross` | 同上 |
| **位置判斷** | `k_above_50`, `d_above_50` | 1 或 0 |
| | `k_above_d`, `dif_above_dea` | 1 或 -1 |
| | `dif_above_zero`, `dea_above_zero` | 1 或 -1 |
| **交叉後計數** | `dif_up/down_since_cross` | 金叉/死叉後持續幾根 |
| | `k_up/down_since_cross` | 同上 |
| | `d_up/down_since_cross`, `j_up/down_since_cross`, `dea_up/down_since_cross` | 同上 |
| **均高/均低** | `avg_high_buy` | high > prev HA-high → 1 |
| | `avg_low_sell` | low < prev HA-low → 1 |

## 七、安全機制

| 層級 | 機制 | 說明 |
|------|------|------|
| **IB 帳戶** | Paper Trading | `TRADING_MODE=paper`, port 4002 |
| **下單數量** | `default_shares=10` | 每筆約 $2,500（TSLA），可 API 調整 |
| **持倉上限** | `max_total_positions=5` | 最多 5 個策略同時持倉 |
| **重複防護** | `_inflight` set | 同一策略不會同時有兩筆訂單 |
| **止損保護** | `price_exit` | 預設 5% stop-loss，每根 bar 檢查 |
| **止盈保護** | `price_exit` | 預設 5% take-profit |
| **全域開關** | `config['enabled']` | `POST /api/trading/config` 即時停用 |
| **IB 斷線** | Queue 保留 | 未連線時 order 放回 Queue 等重連 |
| **策略 reload** | 狀態保留 | reload 不中斷進行中的策略狀態 |
| **線程安全** | `threading.Lock` | 所有共享狀態用 Lock 保護 |
